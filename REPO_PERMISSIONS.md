# GitHub 저장소 권한 가이드

## 현재 상황

저장소(`dealicious-inc/ssm-web`)에 접근은 가능하지만 설정 메뉴가 보이지 않는 것은 **정상**입니다.

## 권한 등급 설명

### 1. Read (읽기 권한) ✅ 현재 권한

**가능한 작업:**
- ✅ 저장소 코드 읽기
- ✅ 저장소 클론
- ✅ 패키지 설치 (`yarn install`, `npm install`)
- ✅ 파일 다운로드
- ✅ 이슈/PR 보기

**불가능한 작업:**
- ❌ 코드 푸시
- ❌ 설정 변경
- ❌ 멤버 관리

**결론**: 패키지 설치를 위해서는 **이 권한만으로 충분**합니다!

### 2. Write (쓰기 권한)

**추가로 가능한 작업:**
- ✅ 코드 푸시
- ✅ 브랜치 생성/삭제
- ✅ 이슈/PR 생성/수정
- ✅ 위키 편집

**불가능한 작업:**
- ❌ 설정 변경
- ❌ 멤버 관리

### 3. Admin (관리 권한)

**추가로 가능한 작업:**
- ✅ 저장소 설정 변경
- ✅ 멤버 초대/제거
- ✅ 저장소 삭제
- ✅ 모든 관리 기능

**패키지 설치에는 불필요**: Admin 권한은 패키지 설치와 무관합니다.

## 패키지 설치에 필요한 권한

### 최소 요구사항

1. **저장소 접근 권한**: Read 권한 (현재 보유 ✅)
2. **Personal Access Token**: `repo` scope (설정 완료 ✅)

### 확인 방법

```bash
# 1. 저장소 접근 확인
# GitHub 웹에서 https://github.com/dealicious-inc/ssm-web 접근 가능한지 확인
# ✅ 접근 가능 = Read 권한 있음

# 2. 토큰 및 권한 확인
./check-repo-access.sh

# 3. 패키지 설치 시도
./install-via-git-credentials.sh
```

## FAQ

### Q: 설정 메뉴가 보이지 않아도 패키지를 설치할 수 있나요?

**A: 네, 가능합니다!**
- 설정 메뉴는 Admin 권한이 필요합니다
- 패키지 설치에는 Read 권한만 있으면 됩니다
- 현재 권한으로 충분합니다

### Q: 권한을 높여야 하나요?

**A: 필요 없습니다.**
- 패키지 설치를 위해서는 현재 권한(Read)으로 충분합니다
- Admin 권한은 저장소 관리자가 부여해야 합니다
- 패키지 설치와는 무관합니다

### Q: "You don't have access to repository options" 메시지가 나와요

**A: 정상입니다.**
- 이 메시지는 설정 메뉴 접근 권한이 없다는 의미입니다
- 저장소 읽기에는 문제가 없습니다
- 패키지 설치에는 영향이 없습니다

```bash
# 1. 저장소 접근 권한 확인
./check-repo-access.sh

# 2. 패키지 설치
./install-via-git-credentials.sh
```

## 권한 요청이 필요한 경우

만약 패키지 설치가 실패한다면:

1. **저장소 관리자에게 문의**
   - 저장소 접근 권한 확인
   - 조직 멤버십 확인

2. **조직 관리자에게 문의**
   - `dealicious-inc` 조직 멤버십 요청
   - 저장소 접근 권한 요청

하지만 일반적으로 Read 권한만 있으면 패키지 설치가 가능합니다!
